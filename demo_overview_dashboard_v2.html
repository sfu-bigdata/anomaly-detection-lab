How Git Tracks Your Work
💻 Your Machine (Local)
☁️ GitHub (Remote)
📝

Untracked / Modified

Files you've created or changed. Git sees them but isn't tracking them yet.

git status
📦

Staging Area

Files marked for the next snapshot. You choose what to include.

git add .
💾

Local Repository

A saved snapshot with a message. Your project's history lives here.

git commit -m "msg"
☁️

GitHub (Remote)

Your work backed up in the cloud. Teammates can pull your changes.

git push

🔑 Key Concepts in Today's Demo

🔄

Git — Version Control

Cloud-based file management. Tracks every change. Enables collaboration. Core loop: add → commit → push.

📁

Organize Deliberately

Noble's rules: name files by content, group similar things. src/ for code, data/ for data, reports/ for outputs.

🚫

.gitignore — What NOT to Push

Virtual environments, caches, secrets, raw PII data. If it can be regenerated or is sensitive, exclude it.

📦

Virtual Environment

Isolates your project's Python packages. Share requirements.txt so anyone can recreate it.

🎯

Use Only Signal Features

If a column doesn't help answer your question — PII, unused IDs, anything — leave it out. Drop it from the CSV, or use usecols= when loading.

🤖

AI-Assisted Workflow

Use Codex to scaffold projects, write code, interpret results. Know what to ask & how to verify.
